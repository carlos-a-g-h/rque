rQUE

How data is stored

In rQUE, data is stored in a large hashmap, where each key is a group name and each value is the content of the group

{
'group 1' : [
['thing1'] , ['thing2','bonus']
] ,
'group2' : [
['thing1']
] ,
'another group' : [
['headname1','data'] , ['headname2','data'], ['headname3','data','more data']
]
}

Groups are lists and each group name is unique
Each item inside a group is a list with the first index being the head of the item
2 or more items in the same group cannot have the same head, and this is checked automatically by rQUE before adding new items to a group

API usage

GET /help
Desc.: This help

GET /
Desc.: Always gives 200
Res.: (200): {}

GET /all
Desc.: Recovers a list of existing group names
Res. (JSON, 200): {'result':['name1','name2',...,'nameN']}
Res. (JSON, 4xx): {}

GET /sel/{name}
Desc.: Recovers all the items of the specified group
Res. (JSON, 200): { 'group' : [ ['thing1',...,'qwe'] , ['thing2',...,'rty'] , ... , ['thingN',...,'uio'] ] }
Res. (JSON, 4xx): {}

GET /sel/{name}/{index}
Desc.: Recovers a selected item from a group by its index
Res. (JSON, 200): {'item':['thing','content',...,'qwe'] }
Res. (JSON, 4xx): {}

GET /sel/{name}/{index}/{qtty}
Desc.: Recovers a slice of a group, using a starting index and a quantity (range selection). Returns 200 if it recovered at least one item
Res. (JSON, 200): { 'slice' : ['thing1',...,'tail'] , ['thing2'] , ['head','data','more'] }
Res. (JSON, 4xx): {}
NOTE: If you set quantity as 0, the item in the specified index and the rest of the items after the selected index will be chosen

POST /add/sin
JSON {'name':'some group','item':['head','content',...,'tail']}
Desc.: Adds a new item to the bottom of an existing group (yes, it's like a queue). Returns 200 if successful
NOTE: If the group does not exist, it will be created automatically
NOTE: If the new item to add matches the head of an existing item, the new item will not be added
Res. (JSON, any): {}

DELETE /all
Desc.: Deletes all groups. Returns 200 if successful
WARNING: This is dangerous
Res. (JSON, any): {}

DELETE /sel/{name}
Desc.: Delete a specific group and all of its content. Returns 200 if successful
Res. (JSON, any): {}

DELETE /sel/{name}/{index}
Desc.: Select by index a specific item in a specific group and delete that item. Returns 200 if successful
